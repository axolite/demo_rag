# nRF Connect SDK — Bare Metal (nrf-bm) reference

This project keeps a **local clone of `sdk-nrf-bm`** as the offline source of truth for
the nRF Connect SDK Bare Metal documentation, headers, and samples. Prefer searching this
local clone over fetching the live docs site.

- Live docs (rendered): https://nrfconnectdocs.nordicsemi.com/ncs-bm/latest/nrf-bm/index.html
- Source repo: https://github.com/nrfconnect/sdk-nrf-bm
- Local clone: `sdk-nrf-bm/` — pinned at commit `894fa18` (2026-06-15)
- Refresh with: `bash refresh-docs.sh`

## Where to look

| You need…                | Look here                                                        |
|--------------------------|------------------------------------------------------------------|
| Narrative / guides       | `sdk-nrf-bm/doc/nrf-bm/**/*.rst` (87 pages; start at `index.rst`)|
| **API reference**        | `sdk-nrf-bm/include/bm/**/*.h` — Doxygen comments are the API docs|
| Kconfig options          | `sdk-nrf-bm/**/Kconfig*`                                          |
| Working example code     | `sdk-nrf-bm/samples/**` (30 samples — see list below)            |
| Library implementations  | `sdk-nrf-bm/lib/`, `sdk-nrf-bm/subsys/`, `sdk-nrf-bm/drivers/`    |

The rendered API site is generated by Doxygen from the headers via `doxygengroup`
directives in `doc/nrf-bm/api/api.rst`. The headers in `include/bm/` are the authoritative
API docs — reading them is richer than the scraped HTML.

## Doc entry point

`sdk-nrf-bm/doc/nrf-bm/index.rst` toctree covers: install, memory layout, drivers,
Bluetooth/services/utility libraries, samples, DFU, release notes, nRF5→BM migration,
SoftDevice docs, and the API reference.

## Headers layout (`include/bm/`)

`bluetooth/`, `drivers/`, `fs/`, `mgmt/`, `nfc/`, `shell/`, `softdevice_handler/`, `storage/`

## Samples (`samples/`)

- `bluetooth/`: ble_bms, ble_cgms, ble_hids_keyboard, ble_hids_mouse, ble_hrs,
  ble_hrs_central, ble_lbs, ble_nus, ble_nus_central, ble_pwr_profiling,
  ble_radio_notification, hello_softdevice, peripheral_nfc_pairing
- `boot/`: mcuboot_recovery_entry, mcuboot_recovery_retention
- `nfc/`: record_text_t2t, record_text_t4t
- `peripherals/`: buttons, leds, lpuarte, ppi, pwm, radio_test, timer, uarte
- `subsys/`: fs, shell, storage

## How to search

```bash
# Find an API by group/symbol
grep -rn "@defgroup ble_nus" sdk-nrf-bm/include
grep -rn "ble_nus_data_send" sdk-nrf-bm/include sdk-nrf-bm/lib sdk-nrf-bm/subsys

# Find narrative docs on a topic
grep -rln "DFU" sdk-nrf-bm/doc/nrf-bm

# Find a Kconfig option
grep -rn "config BM_" sdk-nrf-bm
```
